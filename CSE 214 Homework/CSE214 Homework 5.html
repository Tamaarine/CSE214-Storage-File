In this assignment, you will be modeling an ecological pyramid using a ternary (3-child) tree data structure. Directly below is an example of an ecological pyramid, also called a food pyramid, that your ternary tree will be able to recreate. Please notice that each node represents a single species. In the diagram, animals are displayed with black text and arrows, and plants are displayed with green text and arrows. Animals can be both predators and prey, while plants are always prey and cannot predate on other organisms. In the context of the ternary tree, parent nodes are the predators of their children. The root of the tree is called the apex predator.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 


Understanding ecological pyramids and food patterns are essential in biological research to understand the concepts of biomass and energy transfer. 1.     OrganismNode

Write a fully-documented class named OrganismNode that will represent a single node in your tree, and each node will represent a single species (either a plant or an animal). OrganismNode objects will have three OrganismNode references (left, middle, and right). These three references are the references for children. The class should also include a String name variable for the name of the organism’s species.

The class should also have a boolean value isPlant that will differentiate between animal nodes and plant nodes.

Finally, the class should have two more boolean values, isHerbivore and isCarnivore, that will restrict an animal’s diet. An animal node can be either an herbivore (eats only plants), a carnivore (eats only other animals), or an omnivore (eats both plants and animals). These values should properly reflect true or false depending on whether this node can prey on plants and/or other animals. For example, for omnivores, both values should be true.

PLEASE NOTE: For the sake of this assignment, no plant exists that can consume another organism, and thus, no plant node can have any children. Also, isHerbivore and isCarnivore should always be false for all plants.

2. OrganismTree

This class will implement the ternary tree of OrganismNode objects, and allow various operations such as insertion, removal, printing, moving the cursor reference, and more. This class will contain two OrganismNode references: root and cursor. root references the root of the tree (which is the apex predator), and cursor will reference the node that the user wants to be at. o    Brief:

§  Removes the child node of cursor with name name, and properly shifts the deleted node’s other siblings if necessary. If the deleted node has any descendants, those nodes are deleted as well.

o    Parameter:

§  name The name of the node to be deleted.

o    Precondition:

§  name references a direct child of the cursor.

o    Postconditions:

§  IllegalArgumentException: Thrown if name does not reference a direct child of the cursor.
