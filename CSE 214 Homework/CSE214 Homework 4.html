Encryption is an integral part of cybersecurity and security in general, being used to hide information through some form of obfuscation of the original information. Though modern techniques utilize far more complex forms of encryption, the Playfair Cipher is still effective at masking information and presents a difficult problem to crack without its key. The Playfair Cipher was invented in 1854 by Charles Wheatstone and uses symmetric encryption. This technique takes pairs of letters (often referred to as bigrams or digrams) and encrypts them. This creates a more difficult encyption than simple substitution ciphers which are vulnerable to frequency analysis. The Playfair Cipher uses a 5-by-5 table that contains either a keyword or phrase which acts as the encryption key.

In this assignment, you will execute Playfair Cipher encryption using queues. These will be utilized to transform a sentence or phrase submitted by the user into bigrams. Once these bigrams are created, the set will be encrypted based off a Playfair Cipher key table. The final product will be presented to the user. The user can also request decryptions based off the key used by the current iteration of the program.

ABOUT THE PLAYFAIR CIPHER

Playfair Cipher Keys have a few rules:

  • The keyword or phrase is placed in the key, dropping any duplicate letters.
  • The remaining spaces are filled in alphabetical order, omitting "J" to make the alphabet fit.

The generation of a key with a phrase would occur as follows:

    Phrase: PLAYFAIR EXAMPLE

  1. The phrase is placed into the key in the order the characters appear with duplicates removed.

    2. Key: PLAYFIREXM

  2. The remaining alphabet letters (excluding "J") are then appended to the key to bring the key length to 25.

    3. Key: PLAYFIREXMBCDGHKNOQSTUVWZ

  3. These characters are then arranged in a 5x5 table in the order which they appear.

    4.  P 
     L 
     A 
     Y 
     F
    I R E X M
    B C D G H
    K N O Q S
    T U V W Z

Once the key is made, the data is ready to be encrypted. The sentence must first be broken down into bigrams. In the example "Hello World", the phrase will be broken into "HE LL OW OR LD".

There are four rules to Playfair Cipher encryption:

  • If two letters in a bigram are the same (or there is only one letter), an "X" is added after the first letter and a new pair is formed.
    • Ex. "HE LL OW OR LD" -> "HE LX LO WO RL DX"

  • Letters of a bigram appearing in the same row of the key table are replaced with the letters to their immediate right, respectively. You will need to wrap around to the left if a letter is on the far right side of the table.
    • Ex. "EX" -> "XM" 
     P 
     L 
     A 
     Y 
     F
    I R E→ X→ M
    B C D G H
    K N O Q S
    T U V W Z


  • Letters if a bigram appearing in the same column of the key table are replaced with the letters immediately below them, respectively. You need to wrap around to the top of the column if a letter is at the bottom of the table.
    • Ex. "DE" -> "OD" 
     P 
     L 
     A 
     Y 
     F
    I R E↓ X M
    B C D↓ G H
    K N O Q S
    T U V W Z


  • Letters of a bigram not appearing in the same row or column are replaced with the letters on the same rows, respectively, but at the opposite corners of the rectangle formed by the positions of the original letters.
    • Ex. "HI" -> "BM" 
     P 
     L 
     A 
     Y 
     F
    I R E X M
    B C D G H
    K N O Q S
    T U V W Z

Decryption of the Playfair Cipher is executed by performing these rules in reverse with the proper key.

For more information about the Playfair Cipher, refer to this Wikipedia article.

GENERAL ALGORITHM

For encryption: 

    create a queue p to hold the plaintext bigrams
    for each pair of letters in input string s
        create a bigram b and enqueue b in p
    create a queue e to hold the encrypted bigrams
    while p is not empty
        dequeue bigram b from p
        encrypt bigram b
        enqueue b in e
    return e
For decryption: 
    create a queue d to hold the encrypted bigrams
    for each pair of letters in input string s
        create a bigram b and enqueue b in d
    create a queue p to hold the plaintext bigrams
    while d is not empty
        dequeue bigram b from d
        decrypt bigram b
        enqueue b in p
    return p
To print out a Phrase: 
    while phrase is not empty
        dequeue bigram b
        print b
As an example, consider encoding the text "Hello World" with the key "Playfair example": The string "Hello World" is split into Bigrams and inserted into a Queue: 
    Plaintext Queue: (H,E) (L,X) (L,O) (W,O) (R,L) (D,X)
                       ^                             ^
                     front                          rear
These bigrams are then dequeued one at a time, encypted, and enqueued into another queue containing the encrypted bigrams: 
    First step:
    (H,E) is dequeued from the plaintext queue, encrypted, and enqueued into the encrypted queue
    Plaintext Phrase: (L,X) (L,O) (W,O) (R,L) (D,X)
    Encrypted Phrase: (D,M)

    Next step:
    (L,X) is dequeued from the plaintext queue, encrypted, and enqueued into the encrypted queue
    Plaintext Phrase: (L,O) (W,O) (R,L) (D,X)
    Encrypted Phrase: (D,M) (Y,R)

    Next step:
    (L,O) is dequeued from the plaintext queue, encrypted, and enqueued into the encrypted queue
    Plaintext Phrase: (W,O) (R,L) (D,X)
    Encrypted Phrase: (D,M) (Y,R) (A,N)

    ...

    Final step:
    (D,X) is dequeued from the plaintext queue, encrypted, and enqueued into the encrypted queue
    Plaintext Phrase: EMPTY
    Ecnrypted Phrase: (D,M) (Y,R) (A,N) (V,Q) (C,R) (G,E)
